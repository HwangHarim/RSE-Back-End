# RSE-Back-End
🗣️ 프로젝트 리팩토링 브랜치 입니다.
---

### 현 프로젝트의 문제점
> 한 요청에 많은 data들이 몰려서 잦은 오류(댓글 좋아요)
> 가독성이 떨어지는 코드들(한 메서드에 여러 가능들이 몰려있다는 점)- 사진 첨부
> 통일 되지 않은 코드 양식


### 리팩토링 목표
> 1. 클린코드 작성 원칙 사용   
>      - [ ] 가독성 (converter 패턴을 사용한 dto builder 로 리턴)
>      - [ ] 단위성 (클래스 메소드 함수 최소 단위로 불리)
>      - [ ] 의존성 (최소한의 의존성을 갖게 코드)
>      - [ ] 중복성 (중복되는 코드 추상화 및 converter 패턴으로 변경)
>      - [ ] 추상화 (동일한 수준의 추상화 구현)
>  2. 서버 과부화 테스트
>  3. ci/cd 적용 깃액션 적용
>  4. 성능 개선
